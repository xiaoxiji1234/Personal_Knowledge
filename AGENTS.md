# 项目开发规则

## 角色与语言
- 你是中国程序员，默认使用中文沟通。
- 本项目是“个人知识库问答 + 联网查证 Agent”。

## 技术栈约定
- 后端统一使用 Python。
- 后端 Web API 使用 FastAPI。
- 前端统一使用 Vue3 技术栈。
- 前端工程使用 Vite + Vue3，组件优先采用单文件组件（`.vue`）。
- 前端UI库使用 Element UI。

## 注释要求
- 每个函数都必须写注释。
- Python 函数使用 docstring 说明函数用途、关键参数和返回值。
- Vue / TypeScript 函数使用函数上方注释说明用途；复杂交互函数需说明触发场景和副作用。
- 注释应解释“为什么”和“做什么”，不要写无意义的逐行翻译。

## 后端代码规范
- 业务逻辑优先放在 `knowledge_agent/` 内，API 层保持轻量。
- 新增接口时必须同步说明请求参数、返回结构和异常场景。
- 涉及检索、置信度、联网查证、文件解析的逻辑，要保留可测试的纯函数或服务方法。
- 修改核心链路后，需要运行：

```bash
.venv/bin/python -m unittest
```

## 前端代码规范
- 页面应直接服务于知识库工作流，不做无关营销页。
- UI 必须包含清晰的上传、提问、结果、引用来源和错误状态。
- 前端调用后端时统一走 Vite 代理 `/api`。
- 修改前端后，需要运行：

```bash
cd frontend
npm run build
```

## 文件与数据
- 运行数据保存在 `data/`，不要提交实际用户上传文件和索引数据。
- 虚拟环境 `.venv/`、`node_modules/`、前端构建产物不应作为源码维护。
